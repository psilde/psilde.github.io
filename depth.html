Building a Valorant tracker application for the Australian competitive scene

A no-login standalone desktop application that surfaces live lobby intelligence before a Valorant match starts. Depth operates as a comprehensive tracker, actively preventing high elo players from getting repeatedly wintraded, matched against known cheaters, or queued with smurfs. It runs alone to ensure maximum efficiency with zero performance overhead. Built on Tauri and Rust, used daily by players across Australian Immortal+ lobbies.

Origin

Why I built it

Valorant's competitive scene in Australia is small, especially at the highest ranks. You end up playing with and against the same people over and over. Before a match even starts, there's already a lot worth knowing about the people in your lobby: who's likely better than their rank suggests, or who has a history of breaking the game's rules. That information exists, the game just doesn't show it to you.

Most players were relying on a tool called ValRadiant to get this kind of information. It worked, but it came with real downsides: security concerns, the risk of getting banned, and enough background overhead that competitive players, the type who close every unnecessary app just to squeeze out extra performance, didn't want to run it. I wanted something that opened instantly, sat quietly in the background, and showed you exactly what you needed the moment a match was about to start. Nothing more.

I started building Depth because I was frustrated with the existing options. I finished it because people in the community kept asking when it would be ready.

Architecture

How the no-login approach works

The most common question I get: how does Depth read my game data without asking for my password? This was the design decision I thought hardest about, and the answer is grounded in how the Riot Client actually works at a system level.

When Valorant is running, the Riot Client exposes a local REST API on your machine, accessible only from localhost. To authenticate against it, the client writes a lockfile to a known path on disk containing a randomly-generated port number and a one-time basic-auth password:

Lockfile path
%LOCALAPPDATA%\Riot Games\Riot Client\Config\lockfile
Lockfile format
// name:pid:port:password:protocol
RiotClient:12345:52148:abc123xyz...:https

Depth reads this file, authenticates against the local client, and retrieves your session data: your PUUID and server shard. From there, it queries Riot's Player Data (PD) API using standard Riot authentication headers derived from the local session. Your Riot credentials are never touched by Depth. They never leave your machine. There's nothing to leak.

I chose this approach deliberately. Asking players to hand over their Riot credentials to a third-party app is a significant trust ask, especially in a community that's seen account theft before. By building on the lockfile API, Depth can provide the same data without ever touching user information, and players can verify this themselves by looking at what the app actually does.

Design decisions

Why Tauri over Electron, and why it matters for this audience

Performance was the non-negotiable constraint from day one. Competitive Valorant players close Discord, their browser, and anything else that breathes RAM before queuing. A companion app that costs FPS doesn't get used.

Electron, the obvious choice for cross-platform desktop apps, bundles a full Chromium browser. That's roughly 150–300MB of baseline memory overhead before your app does anything. For a tool that just displays some JSON, that's absurd.

Tauri uses the operating system's native WebView (WebView2 on Windows, which is already installed as part of the OS) and a Rust backend. The Depth installer is under 10MB. Runtime memory usage is a fraction of what Electron would cost. The Rust backend handles all API calls and data processing with no garbage collector overhead. It either works or it doesn't.

"I've enabled Fast mode on my PC and depth is still the one thing that never gets removed from the startup list. It's genuinely lighter than Discord notifications."

Community member, Depth Discord

The other decision: standalone application architecture. Depth is perfectly optimised by just being a desktop application that runs alone. This removes unnecessary performance overhead, is far more efficient, and guarantees players don't drop frames during critical moments while still having full access to tracker data.

Engineering challenge

Hitting API limits at scale, and how I solved it

When Depth had a few users, everything ran fine. As the user base grew, a problem emerged: every lobby load triggers API calls for up to 10 players simultaneously: rank data, match history, agent statistics. At a few thousand concurrent users during peak hours, that's tens of thousands of calls per minute hitting Riot's PD API.

The API started rate-limiting requests. Users were seeing incomplete lobby data. Some players would load, others would show as unknown. The root cause was clear: too many redundant calls for the same players. In a small ranked scene, you frequently queue against the same opponents you faced an hour ago.

The fix was a server-side caching layer in the Rust backend. Player data is cached with a short TTL, typically a few minutes. Long enough that repeated lookups within the same session are served from cache, but short enough that rank changes propagate quickly.

The effect was immediate. API error rates dropped significantly and lobby load times improved, because cache hits return in microseconds rather than waiting on a round-trip to Riot's servers. The caching layer also made the app more resilient to Riot API outages: degraded data is better than no data.

Community & feedback

Responding to users: the dodge problem

Depth has an active Discord server where users report bugs, request features, and give feedback. One recurring complaint that took me time to properly understand: the dodge workflow was broken.

In Valorant, if a player wants to leave a lobby before the match starts (taking a rank penalty to avoid a bad match or dodge known cheaters), they typically close the game client. The problem: Depth would cache the previous lobby state. When the player relaunched and queued again, Depth was still showing data from the old lobby, requiring a full app restart to get accurate data.

A user opened a ticket explaining the exact sequence: dodge → relaunch → requeue → stale data. Initially I'd overlooked this edge case because it seemed rare. But in the competitive community, dodging is common, especially at high elo where a single bad match can cost significant RR.

The fix was straightforward once I understood the problem: poll the Riot Client's lobby state endpoint and invalidate the cache whenever the lobby ID changes. If a player dodges, relaunches, and finds a new lobby, Depth detects the new lobby ID and fetches fresh data automatically. No restart required.

Delivery

Auto-updates and CI/CD

One of the things that distinguishes a real product from a side project is how updates get delivered. I integrated Tauri's auto-updater early so that pushing a fix doesn't require users to manually download a new installer.

The release pipeline runs on GitHub Actions: on every tagged release, CI builds and signs the Windows installer, generates the update manifest, and publishes both to GitHub Releases. On startup, Depth checks the releases endpoint. If a newer version is available, it prompts the user and applies the update in the background. The next launch runs the new version.

This meant that when I shipped the fix for the dodge bug, all active users were on the new version within 24 hours, without me having to tell anyone to update.

Honest limitations

What doesn't work perfectly yet

  • API dependency: Depth relies on Riot's internal PD API, which is undocumented and subject to change without notice. Riot has broken third-party tools before by rotating endpoints or changing auth flows. Maintaining compatibility requires monitoring the community and responding quickly.
  • Windows only: The lockfile-based auth approach works differently on macOS. A macOS port is planned but not yet shipped.
  • Private profiles: Players who have enabled Riot's privacy settings hide their match history and rank from the API. Depth shows what's available. For private accounts, some fields are simply blank.
  • Initial load at peak: Despite caching, very large concurrent spikes (tournament nights, patch days) can still produce occasional slow loads. This is an ongoing area of improvement.
Tech stack

What it's built with

  • Tauri + Rust: System-native desktop app framework with a Rust backend. Handles the lockfile reading, local client auth, PD API calls, caching, and IPC to the frontend.
  • React + TypeScript: Frontend UI rendered in the native WebView. Communicates with the Rust backend via Tauri's typed invoke() bridge.
  • reqwest: Async HTTP client in Rust for all API communication.
  • GitHub Actions: CI/CD pipeline that builds, signs, and publishes Windows installers on tag.
  • Tauri Updater: Auto-update plugin that checks GitHub Releases for new versions on startup.
